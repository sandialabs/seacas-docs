37/*!
38 * This set of classes provides a thin wrapper around the exodusII
39 * internals. It supplants several of the exodusII API calls in
40 * order to avoid ncredef calls which totally rewrite the existing
41 * database and can be very expensive. These routines provide all
42 * required variable, dimension, and attribute definitions to the
43 * underlying netcdf file with only a single ncredef call.
44 *
45 * To use the application must create an Internals instance
46 * and call the Internals::write_meta_data() function. This
47 * function requires several classes as arguments including:
48 * <ul>
49 * <li> Mesh -- defines mesh global metadata
50 * <li> Block -- defines metadata for each block
51 * <li> NodeSet -- defines metadata for each nodeset
52 * <li> SideSet -- defines metadata for each sideset
53 * <li> CommunicationMetaData -- global metadata relating to
54 * parallel info.
55 * </ul>
56 *
57 * Calling Internals::write_meta_data(), replaces the
58 * following exodusII and nemesis API calls:
59 * <ul>
60 * <li> ex_put_init(),
61 * <li> ex_put_elem_block(),
62 * <li> ex_put_node_set_param(),
63 * <li> ex_put_side_set_param(),
64 * <li> ne_put_init_info(),
65 * <li> ne_put_loadbal_param(),
66 * <li> ne_put_cmap_params(),
67 * </ul>
68 */
