35/*!
36 * This set of classes provides a thin wrapper around the exodusII
37 * internals. It supplants several of the exodusII API calls in
38 * order to avoid ncredef calls which totally rewrite the existing
39 * database and can be very expensive. These routines provide all
40 * required variable, dimension, and attribute definitions to the
41 * underlying netcdf file with only a single ncredef call.
42 *
43 * To use the application must create an Internals instance
44 * and call the Internals::write_meta_data() function. This
45 * function requires several classes as arguments including:
46 * <ul>
47 * <li> Mesh -- defines mesh global metadata
48 * <li> Block -- defines metadata for each block
49 * <li> NodeSet -- defines metadata for each nodeset
50 * <li> SideSet -- defines metadata for each sideset
51 * <li> CommunicationMetaData -- global metadata relating to
52 * parallel info.
53 * </ul>
54 *
55 * Calling Internals::write_meta_data(), replaces the
56 * following exodusII and nemesis API calls:
57 * <ul>
58 * <li> ex_put_init(),
59 * <li> ex_put_elem_block(),
60 * <li> ex_put_node_set_param(),
61 * <li> ex_put_side_set_param(),
62 * <li> ne_put_init_info(),
63 * <li> ne_put_loadbal_param(),
64 * <li> ne_put_cmap_params(),
65 * </ul>
66 */
