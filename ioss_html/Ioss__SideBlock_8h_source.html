42 /**
43 *
44 * For externally defined sidesets/sideblocks, attempt to provide
45 * the sideblock name that will be generated if the database is
46 * read and the sideblocks are generated from the sideset at the
47 * read phase. Since sideblocks are not explicitly stored on some
48 * of the database types (e.g. exodus), the IOSS code generates
49 * the sideblocks from the sidesets when reading the database. We
50 * want to maximize the possibility that the same sideblock names
51 * will be generated at that read step as the application is using
52 * for sideblocks that it generates internally to be output to a
53 * restart file that is later read...
54 *
55 * \param[in] sideset_name The name of the sideset that this sideblock will be a member of.
56 * \param[in] block_or_element Depending on the `SurfaceSplitType`
57 * behavior for this database, this is either the name of the
58 * element block that the sideblock is applied to
59 * (SPLIT_BY_ELEMENT_BLOCK) or the topology name of the elements
60 * that the sideblock faces are part of (SPLIT_BY_TOPOLOGIES) or
61 * "UNKNOWN" if mixed topology (SPLIT_BY_DONT_SPLIT)
62 * \param[in] face_topology_name The name of the topology of the
63 * sideblock faces. "UNKNOWN" if not homogeneous.
64 * \returns The generated sideblock name.
65 */
66 static std::string generate_sideblock_name(const std::string &sideset_name,
67 const std::string &block_or_element,
68 const std::string &face_topology_name);
